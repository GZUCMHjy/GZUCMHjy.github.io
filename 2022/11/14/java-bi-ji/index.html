java笔记

1. 静态成员

  • 关键词static

  • 修饰对象

    • 静态常量
    • 静态变量
    • 静态方法
    • 静态类(对内部类的修饰)

  • 应用场景: 多个类需要**共享同一个数据 故用关键词static 修饰 使其方法/属性/常量 固定一个内存空间 让每一个类都可以使用

  • 调用静态成员的格式 ——类名.静态类成员

  • 代码演示

    public class StaticTest{
    static double PI=3.14;//静态常量
    static int id;//静态变量
    public static void method1(){//静态方法
        //do Something
    }
    public void method (){
        
        System.out.println(StaticTest.PI);//调用静态常量
        System.out.println(StaticTest.id);
        StaticTest.method1();
	}
}

  • static作用

    1. 给各类提供共享的数据和方法
    2. 打破虽创建同一个类名但他们却各自独立的情况 改变静态变量/常量 同类调用的时候也是改变后的变量
    3. 总的来说就是static可以让类之间不用继承 也可以一起使用的变量和方法(当然是在类之间没有继承关系但又要共享同一内容的情况下)
    public class StaticTest(){
    static int i=47;
    public StaticTest(){}
    public static void main (String []args){
     		StaticTest staticTest1=new StaticTest();
        	StaticTest staticTest2=new StaticTest();//虽创建同一个类名taticTest 但是jvm分给他们的是不同的内存空间
        	staticTest1.i=60;//static给他们搭起了桥梁 联通了起来 可以使他们共享改变后的i值
        	System.out.println(staticTest2.i);//打印出来是i=60;//而不是i=47;
	}
}

  • 注意:

    1. 静态类成员依旧遵循权限问题 如:静态类方法的私有属性 不能被其他类所调用
    2. ==静态方法体中不能调用非静态类方法和变量==
    3. 静态方法体也不能用this关键词(因为不能调用非静态的属性和方法)
    4. 用static关键修饰的变量必须是 类的属性(就是类的全局变量而非成员方法中的变量[成员方法的变量称作为局部变量])
    //错误例子
public class StaticTest(){
    static int id;
    static double PI;
    public void Test1(){
        //do something
	}
    public static Test2(){
        //do something
	}
    public static void Test3(){
        static int money;//必报错 不能修饰方法的变量
        Test1();//编译器会报错！！！在静态方法中只能调用静态方法！！！
        return this;//不能用this 不过我不明白为什么要写return this 可能为例子才写的吧
    }
}

    2. 常量

    • 关键词 final

    • 修饰对象

      1. 变量(局部变量、全局变量)
      2. 对象
      3. 类
      4. 方法

    • 作用

      1. 对于变量——使其不能被更改 变量变成了常量

        注意:fina关键词定义的变量必须在声明时对其进行赋值操作。

      tip:表示常量的变量要用全大写

      final double PI=3.14;//数学中的常量 不可更改！局部常量
static final double PI=3.14//全局常量 只能在定义时才能赋值 其余不改变！
      1. 对于对象 ——使其只能指向固定的一个对象

      注意:但还是可以改对象的变量值 要想根本不能改变 就必须 用static final修饰

      public final class java1 {
    int id=101;
    public static void main(String[] args) {
        //final 固定指向了java1
        java1 java1 = new java2();//不能指向其他类 会报错 
        java1.id=11;//只固定指向java1 但没限制对该类的变量赋值变值操作
    }
}
class java2{
  .....
}
      1. 对于类——不能被继承 你想想final的作用就是不想被更改 如果能被继承 不就是会遇到子类更改所继承的变量值吗？
      2. 对于方法 ——不能被重写

    3. 内部类

    • 种类

      1. 成员内部类——最常见
      2. 静态内部类——用关键词static修饰的外部类里面的类
      3. 匿名内部类——==重点好吗==
      4. 局部内部类 ——定义在外部类的方法里面

      • 语法格式

        public class OuterClass{//外部类
    ....
        class InnerClass{//内部类
            ....
        }
}

      • 本质——我认为哈 就是套娃(嵌套)类里面还一个类

        • 成员内部类

          1. 内部类可以随意使用外部类的任意的属性和方法(即使是私有的private)

          2. 外部类则不能直接调用内部类的属性 因为内部类的作用域在内部类 作用不到外部类 除非在外部类创建一个内部类

            public class OuterClass{
    InnerClass in=new InnerClass();//在外部类实例化内部类对象(new)
    public class InnerClass{
        public void inf(){
            ...
        }
        int y=0;
    }
    public InnerClass doit(){//这是属于外部类的方法
        in.y=4;//必须在外部类创建内部类 才能在外部类的方法中调用内部类的方法/属性
        //就把内部类当做正常的对象 进行调用
        return new InnerClass();
    }
    public static void main(String []args){
     //要想在方法中获取类的内部类
        //1. 先创建外部类
        //2. 再外部类名class.内部类名class.内部变量名=外部类变量名.new 内部类名class()
        OuterClass out=new OuterClass();
        OuterClass.InnerClass in=out.doit();//靠返回值实例化内部类
      	OuterClass.InnerClass in= out.new InnerClass();//靠外部类直接进行创建new
        //翻译就是 在OuterClass类的InnerClass的变量名 in=在外部类的变量名out里创建内部类InnerClass
    }
}

          3. 当外部类的内部类为私有时，只有该外部类才可以调用使用(前提就是要在外部类创建内部类对线)

            问:那如何才能访问到私有的内部类的方法

            答:接口！

            interface Outerface{
    public void f();
}
class OuterClass{
    private class InnerClass implements Outerface{
        InnerClass(String s){//含参构造器
            ....
        }
        public void f(){//实体化接口方法
            ........
        }
    }
    public OuterClass doit(){
        return new Innerface();
    }
}
public class IntefaceInner{
    public static void main(String []args){
        OuterClass outClass=new OuterClass();
        Outerface outerface=out.doit();//通过接口访问私有内部类
        outerface.f();//调用内部类的方法f();
    }
}

        • 局部内部类

          public class OuterClass{

        }

          1. 外部类的方法外不能直接调用局部内部类的方法/属性

            • 原因：局部内部列的作用域只能作用在自个的方法体中，外部类不能直接调用其内部类的成员属性和方法。

            • 解决方法：

              如果外部类想访问内部类的属性和方法，就必须在外部类的方法中创建内部类对象，从而调用。

              class OuterClass{
    private int age;
    void printAge(){
        class InnerClass{
            void m1(){
                ...
            }
            System.out.println("....");
        }
        InnerClass innerclass=new InnerClass();
    	System.out.println(innerClass.m1());
    }
}

          2. 不能添加访问修饰符，但是可以使用final修饰，达到不会被继承(因为内部类终究还是类)

          3. 内部类可以直接访问外部类的成员属性和方法

          4. 外部其它类不能访问外部类的内部类

          5. 当外部类的成员变量和内部类的成员变量重名时，采用的是就近原则，若想用外部类的成员变量时，采用“外部类名.this.变量名”

            class OuterClass{
    private int n1=100;
    void m1(){
        class InnerClass{
            private int n1=80;
            System.out.println(n1);//这是打印的是内部类的n1，因为就近原则
            System.out.println(OuterClss.this.n1);//这时打印的是外部类的n1成员变量值
        }
    }
}

        • 匿名内部类(定义在外部类的局部位置，比如在方法中)

          • 特点

            1. 本质：类
            2. 特点：没有名字

          • 基本语法

            new 类或接口(参数列表){

            }==; 分号==

            1. 面对对象(也包括抽象类对象)
             class OuterClass{//不能同时有两个或者多个public类，但可以有多个public方法
    public void method(){

        // 匿名类内部类(OuterClass$2) extends father
        //同理面向接口的匿名内部类
        Father father= new Father("jack"){

            public void test(){
                System.out.println("...");
            }

        father.test();
        Aniamle cat=new Animal("TOM",10,male){

            //在匿名内部类中重写继承类Animal的方法test1()
            //如果匿名内部类中继承了抽象类 就必须重写抽象类的抽象方法！！！
            public void test1(){

          1. 使用案例(匿名类作为参数传参)

            public class InnerClass {
    public static void main(String[] args) {
        CellPhone cellPhone = new CellPhone();
        cellPhone.alarmClock(new Bell(){
            //省去了implements，减少了代码的冗长
            //实现接口中的方法ring()

            @Override
            public void ring() {
                System.out.println("懒猪起床了！");
            }

        show show = new show();
        show.show(new Mobile() {
            @Override
            void m1() {
                System.out.println("!!!");
            }

    }
}
//面向接口
interface Bell{
    void ring();
}
class CellPhone{
    void alarmClock(Bell cell){
        cell.ring();//动态绑定 运行类型是匿名内部类
    }
}
//面向抽象类
abstract class Mobile{ 
    abstract void m1();
}
class show{
    void show(Mobile m){
        System.out.println("showMobile!");
    }
}

      4.正则表达式

      1. 标准字符集

      • \d 任意一个数字，0~9中的任意一个
      • \w 任意一个字母或数字或下划线，也就是A-Z,a~z,0-9,_中==任意一个==
      • \s 包括空格、制表符、换行符等空白字符的其中任意一个

      2. 自定义字符集合

      • [ ]方括号匹配方式，能够匹配方括号中任意一个字符

        • [ ^abc ]表示匹配中括号abc字符以外的任意一个字符
        • [ k -f ]表示匹配k到f的任意一个字符

      • { }花括号

        • {n}表达式重复n次
        • {m,n}表达式至少重复m次，最多重复n次
        • {m，}表达式至少重复m次

      5.2 IO流和流的分类

      介绍:

      • I/O是Input/Output的缩写，I/O技术是非常实用的技术，用于处理数据传输，读写文件，网络通讯；在java程序中，对于数据的输入和输出操作都是以”流”的形式进行的。
      • Input(输入):==读取外部数据==(从外部的存储设备中的数据传入到java程序内存中)
      • Output(输出):将程序中的内存数据==传入==到指定的外部存储设备 如:电脑的磁盘，也可以是外部的存储设备，如:U盘，光盘等。

      流的分类

      • 按操作数据单位:**字节流(8bit)二进制文件(比如:图片，视频)，字符类(按字符)**文本文件
      • 按数据流向:输入流，输出流
      • 流的角色:结点流，处理流
